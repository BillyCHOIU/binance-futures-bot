```
        ↓ 选币、进出场位

        ↓
币安 U 本位 API 下单 / 平仓 / 止盈止损
```

## 3. 策略设计

### 3.1 主策略：多币趋势 + 舆情门控

**舆情层（方向闸门）**

- 对每个候选币 / 大盘（BTC 主导）输出：`BULL` / `BEAR` / `NEUTRAL`
- 打分因子：
  - 币安公告：上币/下架/合约调整/安全事件 → 高权重事件
  - X：官方与固定信源列表（非全网乱爬）
  - Reddit：主流子版热帖标题与涨速
  - 媒体标题：监管、ETF、黑客、宏观风险
- 规则：时间衰减、来源权重、同事件去重、币种实体映射
- 输出供交易层：`allowed_side = long | short | both | none`

**指标层（过滤 + 进出场）**

- 流动性：24h 成交额门槛（过滤垃圾币）
- 趋势：EMA 快慢线（如 20/60）方向一致
- 波动：ATR 用于止损距离与仓位
- 入场：趋势同向 + 回踩/突破确认（可配）
- 出场：固定止损 + ATR 移动止盈 / 趋势反转平仓

**为什么不用「纯高胜率网格」当主策略**

- 震荡市好看，单边行情易扛单；与 5～10x 多空、回撤 10～20% 目标不匹配
- 网格可作二期可选模块，且必须带趋势熔断

### 3.2 风控（比策略更重要）

| 规则 | 默认建议（可配置） |
|---|---|
| 单笔风险 | 账户权益的 1%～2% |
| 杠杆 | 默认 5x，上限 10x |
| 最大同时持仓 | 1～3 个（小资金建议 1） |
| 日亏损熔断 | 当日 -3%～-5% 停止新开仓 |
| 总回撤熔断 | 峰值回撤 ≥ 12% 停机（落在 10～20% 区间内） |
| 黑名单 | 可配置；公告重大利空自动临时拉黑 |
| 最小名义 | 尊重币安 min notional；5U 账户可能只能测 API 不能真扫多币 |

**5U 阶段目标**：验证「连上 → 读行情 → 出信号 → 下单/撤单/平仓 → 日志」全链路，**不指望盈利**（手续费 + 最小下单量会吃掉小账户）。

## 4. 系统架构

```
┌─────────────────────────────────────────────┐
│  GUI（CustomTkinter）一键启停 / 填 Key / 日志 │
└───────────────────┬─────────────────────────┘
                    │
┌───────────────────▼─────────────────────────┐
│  Engine 主循环（定时 tick）                    │
│  ├─ MarketData   行情 / K 线                  │
│  ├─ NewsSentiment 舆情方向                    │
│  ├─ Strategy     选币 + 信号                  │
│  ├─ RiskManager  仓位 / 熔断                  │
│  └─ Execution    币安 U 本位下单              │
└─────────────────────────────────────────────┘
          │                    │
          ▼                    ▼
   Binance Futures API    本地 SQLite 日志
   （Testnet / 实盘）      trades / events
```

### 4.1 技术栈（MVP）

| 层 | 选型 | 原因 |
|---|---|---|
| 语言 | Python 3.11 | 本机已有；生态成熟 |
| 交易所 | `ccxt` 或官方 `binance-futures` | U 本位永续标准 |
| GUI | CustomTkinter | Windows 原生窗体，点按钮 |
| 配置 | `config.yaml` + `.env`（Key 不进代码库） | 安全 |
| 舆情 | feedparser / requests + 规则打分 | 先稳后智能 |
| 存储 | SQLite | 零依赖部署 |
| 打包（后期） | PyInstaller | 一键 exe |

### 4.2 新闻 MVP 边界（2～3 天）

**必做：**

1. 币安公告（合约相关）拉取 + 关键词事件分类  
2. 主流媒体 / 聚合 RSS 标题  
3. 规则情绪打分 + 币种映射（BTC/ETH 优先）  
4. 输出 `BULL/BEAR/NEUTRAL` 供策略门控  

**后做：**

1. Reddit（PRAW / 公共 JSON，需限速）  
2. X 固定账号列表（API 成本与合规；或只读公开源）  
3. 更重的 NLP / LLM 摘要（可选，不阻塞主链路）

## 5. 界面与使用流程（非程序员）

1. 双击启动（或 `一键启动.bat`）  
2. 首次：粘贴 API Key / Secret（只勾选合约交易权限，**不要提现权限**）  
3. 选择：模拟盘 Testnet / 实盘（默认建议 Testnet）  
4. 点「启动机器人」→ 看日志与当前方向/持仓  
5. 点「停止」立即停新开仓；可选手动「全部平仓」  

配置页可见：杠杆、单笔风险%、日亏熔断%、回撤熔断%、白名单币、是否启用舆情。

## 7. 产品化注意（以后卖软件）

1. **密钥**：只在用户本机，不上传你的服务器（推荐架构）  
2. **权限**：文档强制要求 API 禁止提现  
3. **合规话术**：工具 ≠ 投资建议；历史回测 ≠ 未来收益  
4. **订阅**：许可证 Key 或账号登录解锁；策略更新走版本发布  
5. **责任边界**：用户自主决策开启；软件故障/滑点/爆仓免责需写清  

---

## 8. 风险与诚实预期

- 合约有杠杆爆仓风险；机器人不能消灭风险，只能约束纪律  
- 无公开「高胜率稳定盈利」圣杯；存活靠风控 + 正期望 + 长期迭代  
- 舆情可被操纵；必须来源权重与事件去重  
- 5U 几乎只能测链路；有效策略验证建议更大资金且小仓位比例  
- 任何上线策略都会衰减，需持续监控
